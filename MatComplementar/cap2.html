2.1 DataFrame e estrutura de séries temporais +
O que é

Uma série temporal é uma sequência de valores medidos ao longo do tempo. No pandas, você armazena isso num DataFrame com uma coluna de tempo (geralmente chamada ds ou Month) e uma ou mais colunas de valor.

  • Timestamp — um instante exato: 2024-01-15 08:30:00
  • Period — um intervalo: "janeiro/2024" (sem hora exata)
  • Colunas-chave — quando há múltiplas séries no mesmo DataFrame, você as identifica por combinação de colunas (ex: Estado + Gênero)
Analogia

Pensa num dataset de sensores IoT do MiRoute: cada leitura tem um timestamp, um ID de sensor e o valor medido. O timestamp é o eixo do tempo, o ID é a chave que separa as séries, e o valor é o que você vai prever.

Padrão básico — como estruturar
# Lendo CSV e já parseando a coluna de tempo
df = pd.read_csv("dados.csv", parse_dates=["ds"])

# Filtrando uma série específica por chave
serie_a = df.loc[df["sensor_id"] == "A01"]

# Agrupando por tempo (somando múltiplas séries)
total = (
    df.groupby("ds", as_index=False)
    .agg({"valor": "sum"})
)

2.2 – 2.3 Time plots e padrões temporais +
Os 3 padrões que você precisa reconhecer
  • Tendência — crescimento ou queda persistente ao longo do tempo. Ex: custo de scripts médicos subindo ano a ano.
  • Sazonalidade — padrão que se repete em período fixo e conhecido. Ex: vendas caem toda segunda, explodem todo dezembro.
  • Ciclo — oscilação sem período fixo, ligada a fatores externos (econômicos, climáticos). Pode durar anos e não tem regularidade previsível.
Como diferenciar ciclo de sazonalidade

Sazonalidade tem calendário: você sabe que vai acontecer em dezembro. Ciclo não tem calendário: você só percebe depois que já aconteceu. Recessão econômica é ciclo. Black Friday é sazonalidade.

O que o time plot revela

Plota valor no Y, tempo no X. É o primeiro gráfico que você faz, sempre. Você consegue ver de cara: tem tendência? Tem sazonalidade visível? Tem outliers? A série é estacionária (oscila em torno de uma média) ou não?

2.4 – 2.5 Seasonal plots e subseries plots +
Seasonal plot

Em vez de ver todos os anos em sequência, você sobrepõe cada ano num mesmo eixo X de meses (Jan-Dez). Cada linha é um ano diferente. Isso deixa claro se o padrão sazonal é estável (linhas parecidas) ou está mudando (linhas divergindo).

Seasonal subseries plot

Divide a série em "fatias" por período (um painel por mês, por dia da semana, etc). Dentro de cada fatia, você plota todos os anos com a média marcada. Serve pra comparar: o padrão de janeiro é muito diferente do padrão de julho? A média de cada mês está crescendo ao longo dos anos?

Quando usar cada um

Seasonal plot → "o padrão sazonal está mudando ao longo dos anos?"
Subseries plot → "qual mês/dia tem maior magnitude sazonal? Ela é estável?"

2.6 Scatterplots e correlação +
Pra que serve aqui

Séries temporais muitas vezes têm covariáveis — outras variáveis que ajudam a explicar o valor que você quer prever. O scatterplot mostra se há relação entre elas. Ex: temperatura vs consumo de energia elétrica.

  • Correlação de Pearson: de -1 (inversa perfeita) a +1 (direta perfeita). Zero = sem relação linear.
  • Cuidado: correlação não é causalidade. E relações não-lineares podem ter correlação baixa mesmo existindo relação forte.
Scatterplot matrix

Quando você tem várias variáveis, plota todas contra todas de uma vez. É uma grade de scatterplots. Você identifica rapidamente quais pares têm relação forte antes de modelar.

2.7 Lag plots +
O que é um lag

Lag 1 = o valor de ontem. Lag 12 = o valor de 12 meses atrás. Um lag plot plota o valor atual (eixo Y) contra o valor de N períodos atrás (eixo X).

Se o plot mostrar uma nuvem sem forma → o passado não explica o presente.
Se mostrar uma linha ou elipse clara → há autocorrelação, o passado tem poder preditivo.

Analogia

Se a temperatura de hoje tem relação com a de ontem (lag 1), faz sentido usar ela como variável preditora. Se a receita de dezembro tem relação com a de dezembro do ano passado (lag 12), o modelo precisa capturar esse lag anual.

2.8 Autocorrelação (ACF) +
O que é

O ACF (Autocorrelation Function) mede a correlação entre a série e ela mesma deslocada por N períodos, para todos os lags de uma vez. O resultado é um gráfico de barras onde cada barra = um lag.

  • Barra acima da linha pontilhada → autocorrelação significativa naquele lag
  • Barras altas decaindo devagar → tendência
  • Picos em lags específicos (ex: 12, 24...) → sazonalidade
  • Todas as barras baixas → ruído branco

Verde = tendência  |  Roxo = pico sazonal  |  Linha pontilhada = limite de significância

Como plotar o ACF em Python
from statsmodels.graphics.tsaplots import plot_acf
import matplotlib.pyplot as plt

# lags=40 mostra os primeiros 40 defasagens
plot_acf(df["valor"], lags=40)
plt.show()

2.9 White noise e validação de modelos +
O que é ruído branco

Uma série onde os valores são completamente aleatórios e independentes. Sem tendência, sem sazonalidade, sem autocorrelação. O ACF mostra todas as barras dentro do intervalo de confiança.

Por que isso importa na prática

Depois que você ajusta um modelo de previsão, você analisa os resíduos (erros do modelo). Se os resíduos parecem ruído branco, o modelo capturou toda a estrutura previsível da série. Se ainda tiver padrão nos resíduos, o modelo está deixando informação na mesa.

Regra prática

Resíduo bom = ruído branco.
Resíduo com padrão no ACF = modelo ruim, precisa melhorar.
